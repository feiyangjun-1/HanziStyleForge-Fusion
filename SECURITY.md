# Security policy

## Supported version

Security fixes are applied to the latest published release.

## Reporting

Report security-sensitive issues privately to the repository owner through the private vulnerability-reporting feature on GitHub when it is enabled. Do not include proprietary fonts, trained weights, generated glyphs or private filesystem paths in a public issue.

Useful reports include the affected version, operating system, exact command, minimal reproduction and impact. Remove font files and other licensed assets before sharing logs or archives.

## Scope

Examples include unsafe archive handling, command injection, unintended file deletion, path traversal, untrusted font parsing problems and accidental inclusion of private artifacts in release packages. Model-quality disagreements and ordinary font-rendering differences are not security vulnerabilities.
